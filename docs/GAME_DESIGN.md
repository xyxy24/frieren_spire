# 《秘法高塔》（暂名）游戏设计文档

> 魔法的像素动画、共享特效模板、资源规格和逐项演出方案见 `MAGIC_VFX_DESIGN.md`；该文档只约束表现，不改变本文中的玩法与数值规则。

| 字段 | 内容 |
|---|---|
| 文档版本 | 0.3 |
| 状态 | 概念基线 / 待原型验证 |
| 类型 | 2D 动作 Roguelike、爬塔、单局构筑 |
| 首发平台假设 | Windows PC |
| 核心体验 | 以灵活移动和魔法搭配突破逐层战斗，在三次 Boss 战中完成一局构筑 |

## 1. 产品愿景

玩家扮演一位原创的长寿魔法使，从高塔底层出发。每一层都是一次短而明确的战斗、事件或资源决策；玩家通过收集魔法、调整三个普通魔法槽与一个 Boss 终极魔法槽、购买物品并组合遗物，逐渐形成独特构筑。击败第三位 Boss 后完成本局挑战。

本项目的视觉气质可参考“冷静、克制、擅长解析魔法的高阶魔法使”，战斗节奏可参考高机动平台动作游戏的精准移动、读招与反击感。课程版本可以使用明确标注来源的参考图与占位素材，重点验证玩法和技术；若以后公开发布，再统一替换需要原创化的角色、场景和音频资产。

## 2. 设计支柱

### 2.1 精准且可读的动作战斗

- 玩家应能理解伤害来源、敌人前摇、命中范围和无敌/受击状态。
- 操作反馈优先于复杂系统数量：移动、攻击、受击、施法必须清晰可靠。
- 难度来自组合与节奏，而不是不可读的随机伤害。

### 2.2 三槽构筑与终极魔法

- 玩家可以学会多种普通魔法，但战斗中只能在三个普通槽中装备三个。
- 每个普通魔法拥有独立冷却时间，选择何时释放构成主要战术决策。
- 击败第一位 Boss 后启用一个独立终极魔法槽；它只能装备已获得的 Boss 强力魔法，不算普通槽，也不能装备普通魔法。
- 所有 Boss 魔法共享终极槽的 `18 秒` 公共冷却；同一场战斗内换装或暂停不能刷新该冷却，新战斗开始时恢复可用。
- 楼层间允许重新配置，鼓励针对敌人与 Boss 调整构筑。

### 2.3 有代价的成长选择

- 普通战奖励、Boss 奖励、商店和事件共同塑造一局构筑。
- 遗物可以提供纯增益，也可以提供“提高伤害但承受更多伤害”之类的风险收益交换。
- 奖励选择应尽量产生不同打法，而不只是数值大小排序。

### 2.4 清晰的逐层节奏

- 一次只加载当前楼层，结束后通过楼梯进入下一层。
- 每层有明确的进入、遭遇、结算和离开阶段；魔法配置是可随时打开的覆盖界面，不是强制阶段。
- 固定进度节点安排 Boss，总计三次；第三次 Boss 战胜利即通关。

## 3. 核心循环

1. 进入由本局种子与楼层编号决定的新楼层。
2. 识别本层类型：普通战斗、事件、商店或 Boss。
3. 完成本层遭遇并获得相应结果。
4. 普通敌人死亡处生成魔法掉落物；玩家靠近并交互后，才打开三个普通魔法候选并选择一个。
5. Boss 死亡处生成强力魔法掉落物；玩家靠近并交互后，才打开三个强力魔法候选并选择一个；第一次获得强力魔法时同时启用终极槽。
6. 奖励自动加入对应的普通或 Boss 已学魔法列表，不自动替换任何装备槽；玩家可随时打开魔法装备栏自行调整三个普通槽和一个终极槽。
7. 返回当前楼层地图，移动到楼梯实体旁并交互，恢复 50% 已损失生命后进入下一层。
8. 击败第三位 Boss 获得胜利；HP 降至 0 则本局失败。

## 4. 单局结构

### 4.1 三幕结构

一局分为三幕，每幕固定 5 层并以一场 Boss 战结束，完整流程共 15 层。Boss 固定出现在第 5、10、15 层；其余楼层由确定性调度在普通战、事件与商店之间选择。

| 阶段 | 内容目标 | 结尾 |
|---|---|---|
| 第一幕：阿乌拉与刽子手 | 暮色城门、破碎市街、审判庭与断头台；教会基础敌人、施法、跳跃换层和奖励选择 | 阿乌拉，验证基础构筑 |
| 第二幕：北部边境 | 雪原关隘、断崖、哨站与冰封堡垒；提高纵向换位和敌人组合压力 | 北境 Boss，检验构筑完整性 |
| 第三幕：一级魔法使选拔 | 林地考试、结界回廊、地下迷宫与评审厅；组合前两幕地图机制与终局资源管理 | 最终考试 Boss，胜利结算 |

每幕的五层节奏基线为：至少两次普通战提供魔法成长，一次商店或事件提供资源决策，剩余一层作为普通战、事件或商店的确定性变化位，第五层固定为 Boss。商店与事件继续使用保底计数，具体顺序不强制写死，以保留重玩变化。

### 4.2 楼层类型

| 类型 | 进入后的主要活动 | 结算 |
|---|---|---|
| 普通战斗 | 击败一个或多个敌人 | 金币 + 普通魔法三选一 |
| 事件 | 与房间中央 NPC 交互后作出选择 | 从现有事件中确定性抽取本局尚未触发的事件；结果在本层持久保存 |
| 商店 | 与房间中央商人交互并使用金币购买商品 | 当前原型分行提供最多 3 本魔法书和 2 件遗物，使用固定价格与确定性库存 |
| Boss | 完成阶段性高强度战斗 | 强力魔法三选一；第三次胜利后通关 |

楼层序列不要求完全随机。Boss 节点必须固定，商店与事件可受保底规则约束，避免长时间不出现关键资源节点。

正式游戏每次启动时生成新的 64 位本局种子；同一局内的楼层、事件、商店、奖励和 Boss 选择仍可由窗口标题显示的种子完整复现。课程自动化测试继续使用固定种子。

商店和事件层统一使用不含平台高低差的纯平地安全房，进入时保持可移动状态且不自动弹出选项。玩家靠近中央 NPC 按交互键打开界面，关闭后仍通过房间后方楼梯离层。商店界面第一行陈列魔导书、第二行陈列遗物，卡牌下显示价格；左侧详情面板随选择同步显示魔法的完整效果、CD、范围，或遗物的完整被动效果。使用 WASD 选择、Enter 购买。商人可重复交互，已购买商品立即从本层库存移除。事件界面显示事件标题与背景概述，卡面下保留选项效果；同一事件每局最多触发一次，且在当前事件层只能结算一次，再次与 NPC 交互只重新显示已保存的选择及效果，不重复应用效果。若本局三个事件均已触发，后续原定事件层改为普通战斗层。

战斗奖励三选一界面必须在确认前同时展示每本魔法的名称、具体效果、冷却时间和权威范围摘要，并明确标出 `U / I / O` 对应关系；玩家不应依赖获得后的装备栏才能理解奖励。

玩家确认奖励后，魔法立即加入对应的已学普通/Boss 魔法列表，但不会自动替换装备槽；随后在当前场景上播放独立的魔法注册演出。演出参考《Alice in Cradle》魔法装置的实际 `EfSpellRegisteration` 流程：新魔法光球悬浮在角色上方，旋转空间方框和带引线的工程信息窗依次完成解析；参考时间轴约第 384 帧起，上升字符、圆环、多边形、放射线和子节点构成的巨大注册网络逐渐覆盖画面，并由白青色过渡到红色；约第 713 帧触发白色十字星爆、80 条放射光线与多批光粒，最后将魔法卡面、名称、完整作用、冷却和范围落版。课程游戏以 `3×` 速度完整播放这些阶段，实际总时长约 5.7 秒；Boss 魔法只改变最终强调色，不改变注册结构。演出期间暂停底层玩法并消费暂停、构筑、移动、施法和交互输入，角色播放一次拾取姿态并停在末帧，不显示移动或跑步动画；开始 0.35 秒后可按 `Space` 跳过演出并直接显示完整结果页，结果页仍只接受 `Enter` 关闭，避免误操作略过魔法说明。关闭后玩家仍需自行打开 `Tab` 将新魔法装备到对应槽位。

## 5. 玩家系统

### 5.1 核心状态

| 状态 | 说明 |
|---|---|
| 当前 HP / 最大 HP | HP 为 0 时本局失败 |
| 金币 | 在商店购买魔法或遗物 |
| 已学会魔法 | 本局中获得、可用于配置的魔法集合 |
| 三个普通魔法槽 | 战斗中可直接使用的三个普通魔法，各自保留独立 CD |
| 终极魔法槽 | Boss 1 后启用，只能装备一个已学 Boss 魔法，使用独立输入并共享 18 秒公共 CD |
| 遗物 | 本局内持续生效的被动修正 |
| 楼层与 Boss 进度 | 决定内容池、难度和胜利条件 |

“永久被动增益”在本版文档中解释为“获得后持续到本局结束”。是否存在跨局永久成长尚未决定。

### 5.2 基础动作假设

最低动作集合为：左右移动、跳跃、基础攻击、冲刺、受击、施放三个已装备普通魔法、施放一个已装备 Boss 终极魔法、与楼梯/商人/事件交互。下落攻击、攀墙、二段跳和空中滞留等能力仍为待验证项。

空间尺度直接沿用当前代码：主角碰撞体为 `42×64 px`，普通攻击从角色身体前缘向外延伸 `58 px`，判定高度 `36 px`、相对角色顶部偏移 `14 px`。移动速度为 `260 px/s`，理论跳高约 `132 px`。所有魔法射程、区域半径、位移距离和关卡间隔都直接使用像素，详细权威表见 `MAGIC_RELIC_SYSTEM_DESIGN.md`。

原“高速移动魔法”重新分类为开局固有能力，不进入奖励池、不占魔法槽。`K` 在 `0.18 秒`内沿起始朝向移动 `150 px`，期间锁定方向且不能执行其他动作，结束后恢复冲刺前的垂直速度。普通冲刺冷却 `0.6 秒`且不提供无敌；黑冲使用独立的 `1.5 秒`充能，充满后下一次冲刺自动升级为黑冲，获得全程无敌并标记穿过的敌人 2 秒，使用后重新充能。充能期间不持续显示粒子；仅在充能从未完成变为完成时，播放一次约 `0.35 秒`、最多 8 个黑紫粒子向角色胸口汇聚的就绪提示，随后立即消失。开局默认已充满时不误播提示。HUD 不显示黑冲充能条或普通冲刺冷却条。黑冲未充满不妨碍普通冲刺。当前碰到世界边界会提前停止，未来实体墙必须接入同一停止规则。原 `L` 防御因试玩中缺少主动使用价值而退役，防守能力改由黑冲、女神加护、防御结界、幻影与石之魔像等构筑承担；稳定 ID `1007` 不复用。

### 5.3 生命恢复

离开楼层时恢复“已损失生命”的 50%，而不是最大生命的 50%。

```text
missingHP = maxHP - currentHP
healing = missingHP * 0.5
newHP = min(maxHP, currentHP + healing)
```

如果 HP 使用整数，取整方式必须全局统一。建议原型先采用向上取整，以减少跨层后仅差 1 点生命的挫败感，最终以测试结果决定。

## 6. 战斗系统

### 6.1 战斗目标

- 输入响应稳定，角色状态明确。
- 敌人攻击具备足够前摇和视觉/音效提示。
- 当前普通敌人原型会主动水平追击玩家，并按“追击 → 前摇 → 有效攻击 → 后摇”循环。前摇和攻击阶段使用不同颜色提示；命中造成水平击退与短暂硬直，同一次攻击只结算一次。
- 伤害、击退、受击硬直和无敌帧由统一规则计算。
- 魔法效果在视觉上能区分伤害、控制、防御、位移和持续区域。
- 玩家每次真正损失 HP 后获得 `0.60 秒`受击无敌；窗口内后续敌方攻击不再扣血，也不重复触发受击。由护盾完全吸收、黑冲或其他无敌效果挡下的攻击不启动这段时间。受击无敌与黑冲无敌是相互独立的状态，均只在战斗时间推进时计时。
- 成功扣血必须同时触发受击动画、短暂闪烁和玩家伤害数字；玩家或敌人扣血均触发目标闪白与按伤害强度缩放的短镜头震动。上述表现只读取权威 HP 变化，不参与伤害判定。
- 实际命中会请求一次短命中停顿：18 点及以下伤害停顿 `0.06 秒`，19～35 点停顿 `0.09 秒`，36 点及以上停顿 `0.13 秒`；玩家受击停顿 `0.075 秒`。同一帧多个命中只采用最长请求，不累加时长。停顿期间战斗模拟、角色动作和技能冷却不推进，但菜单、HUD、闪白、飘字与命中扩散环继续使用真实帧时间；表现结束后不得补算被冻结的时间。

### 6.2 魔法施放

- 三个槽位分别绑定一个输入。
- 每个普通魔法拥有独立内置 CD；当前设计不使用法力值。
- 终极槽使用第四个独立输入，当前键盘默认键为 `R`。
- 终极槽拥有一条权威公共冷却，基础时长为 `18 秒`。施放任意 Boss 魔法都会启动它，战斗中更换 Boss 魔法必须继承剩余时间。
- 每场战斗开始时终极槽可用；战斗结束后清理本场冷却。终极槽锁定或未装备时不接受施法输入。
- 冷却开始时点、施法是否可被打断、切换槽位后是否保留冷却，都必须由魔法规则明确声明。
- UI 必须显示每个槽位的魔法图标、可用/冷却状态和剩余时间。

普通内容池现包含 24 个可收集魔法：`1001～1006`、`1008～1009`、`1011`、`1016～1030`，覆盖直伤、元素联动、反制、控制、召唤、位移、护盾、追踪和聚怪。`1010` 是不能学习或装备的固有冲刺；`1007` 与旧方案 `1012～1015` 已退役且 ID 不复用。普通战斗奖励按“一个伤害类、一个控制/反制类、一个全池随机”生成三选一。各魔法的权威作用范围由战斗领域生成，奖励、商店与配装 UI 直接显示完整效果、冷却和范围。完整数值见 `MAGIC_RELIC_SYSTEM_DESIGN.md`。

### 6.3 伤害管线

建议统一按以下概念顺序计算，具体公式在平衡阶段确定：

1. 攻击或魔法产生基础伤害。
2. 施法者增伤、条件加成和遗物修正。
3. 目标减伤、易伤或护盾修正。
4. 结果取整并扣除 HP。
5. 触发受击、击杀和遗物事件。

当前原型已按“基础伤害 × 来源倍率 × 目标倍率 − 固定减伤 → 统一取整 → 实际 HP 变化”建立集中式解析器。倍率、减伤和具体取整参数仍可在平衡阶段调整，但所有战斗伤害必须经过同一入口。

玩家受击入口在解析前统一检查黑冲/技能无敌与受击无敌，在 `DamageResult::appliedDamage > 0` 后才启动 `0.60 秒`受击无敌并递增受击序列。持续攻击仍可每个逻辑节拍提交独立伤害请求，但落在窗口内的节拍会被明确标记为阻挡；例如每 `0.5 秒`结算的喷火或龙卷不能绕过该规则。

同一公式必须同时供战斗逻辑、测试与 UI 预览使用，UI 不重复实现伤害计算。

## 7. 魔法系统

魔法、遗物、统一状态、构筑原型和首轮数值预算见 `MAGIC_RELIC_SYSTEM_DESIGN.md`。24 本可收集普通魔法、固有冲刺 `1010`、9 本 Boss 魔法、遗物 `4001～4024` 与事件遗物 `4101/4102` 已进入可玩原型；`1007`、`1012～1015` 已退役，`2004/2005/2008` 当前不加入奖励池。团队应在试玩后同步调整代码、测试与该文档。

### 7.1 魔法分类建议

| 类别 | 作用 | 示例方向 |
|---|---|---|
| 直接伤害 | 快速输出、补刀 | 魔弹、短距爆发、贯穿射线 |
| 范围伤害 | 应对群体或区域封锁 | 爆炸、落雷、持续法阵 |
| 控制 | 创造安全窗口 | 减速、冻结、击退、束缚 |
| 防御 | 降低风险 | 护盾、短暂无敌、伤害转化 |
| 位移 | 改变站位并参与连招 | 瞬移、冲刺、牵引 |
| 构筑核心 | 改变其他魔法或资源规则 | 冷却联动、元素标记、风险增伤 |

### 7.2 学会与装备

- 获得魔法后加入“已学会魔法”集合。
- 玩家在奖励结算后可打开配置界面。
- 普通魔法只有放入三个普通槽之一才可施放；Boss 魔法只有放入终极槽才可施放。
- 普通魔法不能放入终极槽，Boss 魔法不能占用普通槽。
- 重复魔法、升级机制和替换规则尚未确定；当前原型排除已学魔法，候选不足三个时自动改发 15 金币。

### 7.3 奖励三选一

- 击败最后一个敌人不会立即打开奖励界面，而是在其死亡位置生成带翻页、悬浮和魔力光晕表现的可交互魔法书；只有玩家进入交互范围并确认后才进入三选一。若玩家在空中完成交互，进入奖励页前必须由玩法层将角色落到当前地面并清除残余移动状态，避免奖励与获取演出冻结半空姿态。
- 战斗施法键与奖励选择键即使复用，也不得在状态切换时自动沿用；持续按住 `U/I/O` 不能跳过掉落物交互。
- 普通战从普通魔法池抽取三个不同候选。
- Boss 战从强力魔法池抽取三个不同候选。
- 候选生成应记录 RNG 种子和楼层，便于重现问题。
- 当候选池不足或玩家已拥有全部候选时，需要金币、升级或重投等奖励替代方案；具体方案待定。

## 8. 遗物系统

遗物在获得后于本局持续生效，主要用于改变玩家决策，而不是只堆叠小数值。

| 类型 | 设计方向 |
|---|---|
| 无条件增益 | 最大生命、基础伤害、金币收益等 |
| 风险收益 | 增伤同时提高受到伤害；低生命时更强 |
| 魔法联动 | 某类魔法命中后缩短另一槽位冷却 |
| 节奏变化 | 进入新楼层获得护盾；无伤结束获得奖励 |
| 经济变化 | 商店折扣、重投、出售或利息机制 |

所有遗物需声明叠加规则、触发时机和上限。可能引起递归触发的遗物必须有防重入策略。

当前遗物内容：

- 两件基础示例：`暗黑龙的角（4001）`使所有伤害提高 20%，但每场战斗前 15 秒受到的实际伤害提高 20%；`苍月草的花（4002）`在战斗开始时自动生成一次催生花田，不要求学习或装备，也不启动槽位 CD 或触发施法遗物。
- 已实现剑之乡事件遗物：`勇者之剑（4101）`使敌人碰撞伤害固定减少 10；`真·勇者之剑（4102）`在玩家实际受到碰撞伤害后，对自身周围 54 px 内所有敌人造成 30 遗物伤害。
- `4001～4024` 共 24 件普通/扩展遗物均已实现，覆盖标记连射、冰火控制、冲刺穿敌与墙击、驭血、召唤、飞行普攻、终极槽与经济路线；权威数值、触发边界、敌人应对关系和构筑示例见 `MAGIC_RELIC_SYSTEM_DESIGN.md` 第 7～8 节。
- 普通遗物不能通过名称直接判断具体魔法，而应读取魔法标签、状态和统一战斗事件；事件专属 `4101/4102` 不进入商店池。

- 阿乌拉战后对话：芙莉莲“自裁吧，阿乌拉”；阿乌拉“这不可能……我竟然……”。战前、首次支配和战后对话期间，整个战斗模拟暂停，按 Enter 逐句继续；战后对话结束后才提交 Boss 胜利结果。
- 进入 Boss 战时先暂停战斗，在屏幕中央淡入并淡出 Boss 名称“断头台阿乌拉”，随后才进入战前对话。对话框必须最后绘制，覆盖楼梯、HUD 和技能槽，但保留半透明背景以隐约显示战场。
- 阿乌拉在战前第三句台词开始前使用 `initial` 形象，此后使用 `idle`；支配使用 `windup` 形象，并在权威判定区内播放八帧紫色天平与锁链特效，前六帧预警、后两帧命中与消散。灵魂断头台把固定框架与运动刀刃拆为两张独立像素素材：框架只贴合权威判定区边缘，刀刃在 0.9 秒预警中悬于顶部，并由程序在 0.18 秒有效阶段贯穿处刑区落到底部；素材不得以整张巨型武器填满判定区。紫红处刑柱与地面法阵继续由程序绘制并直接读取权威范围。“不死大军”是无前摇、无施法过程的场地召唤。阿乌拉死亡后立即消除场上所有其他敌人并进入战后对话，阿乌拉保持 `die` 形象直到对话结束，随后视为战斗胜利。
- 第一幕 Boss 暂时固定为断头台阿乌拉。红镜龙及其吐焰、爪击原型延期，不进入当前版本的楼层生成或 Boss 池；若后续时间允许再恢复并完善。
- Boss 房间只有主 Boss 使用右上方固定 Boss 血条；阿乌拉召唤的无头骑士使用各自头顶的小血条。

#### 第二幕 Boss：雷沃戴

- 默认十五层流程的第 10 层固定生成雷沃戴作为第二幕 Boss。
- 雷沃戴 HP 300，碰撞体 `64×96`，移动速度 `160 px/s`，具有默认碰撞伤害。多个技能同时完成冷却时严格按“上斩、下斩、重击、招架、突进”的顺序选择。
- 上斩与下斩 CD 均为 7 秒、前摇 0.3 秒，在前摇结束瞬间对前方 42 px 造成 20 伤害，并于前方 42 px 处分别生成偏上或偏下的 `32×42` 移动斩击；斩击以 `200 px/s` 前进最多 144 px，命中造成 20 伤害和击退。技能触发射程为最远中心距离 `42+144=186 px` 的三分之二，即 124 px。
- 重击 CD 9 秒、前摇 0.5 秒，对前方 64 px 造成 25 伤害，并在前方 64 px 处生成 `54×72` 巨型剑气；剑气以 `200 px/s` 前进最多 168 px，命中造成 25 伤害和击退。技能触发射程为最远中心距离 `64+168=232 px` 的三分之二，约 154.67 px。
- 招架 CD 9 秒、无前摇，冷却完成后无需玩家进入触发距离即可释放；招架状态持续 1.2 秒。期间第一次被魔法实际命中时，该次伤害降低 50%，招架立即结束并强制释放突进。
- 突进 CD 6 秒、前摇 0.5 秒；仅第二阶段可由 AI 主动选择，冷却完成后无需玩家进入触发距离即可释放，招架反击同样不受阶段和当前冷却限制。突进以 `220 px/s` 向锁定方向移动 144 px；结束时令其他技能立即完成冷却，突进自身进入完整 6 秒冷却。
- 第一阶段 HP 首次将降到 5 以下时，伤害被截断并锁定为 5 HP，战斗暂停进入二阶段对话；对话结束后恢复到最大 HP 的 50%（150 HP），随后解除锁血并进入第二阶段。
- 战前对话：雷沃戴“一个精灵魔法使，你是谁”；芙莉莲“四刀流，就算使用防御魔法，也难以正面接住你的斩击吧”；雷沃戴“我在问你是谁”；芙莉莲“你在消灭害虫时，每次都会报上名号吗”。二阶段开始：雷沃戴“有趣，看来得和你认真打上一场了”；芙莉莲“……”。战败：雷沃戴“是你赢了，精灵”。所有对话期间暂停战斗，战败台词结束后才提交胜利结果。

#### 第三幕 Boss：水镜恶魔

- 默认十五层流程的第 15 层固定生成水镜恶魔作为最终 Boss。水镜恶魔 HP 200，本体无法被玩家直接造成伤害，也不会主动造成伤害；只有本体使用右上方 Boss 血条，复制体使用各自头顶血条。
- 第一阶段开局同时召唤休塔尔克复制体和菲伦复制体。两者全部死亡后，水镜恶魔失去当前一半生命并暂停进入二阶段对话，随后召唤芙莉莲复制体。芙莉莲复制体死亡后水镜恶魔死亡；战后对话结束才提交第三 Boss 胜利。
- 休塔尔克复制体：HP 300，碰撞体 `54×72`，移动速度 `200 px/s`，无碰撞伤害；HP 低于 150 后移动速度提高到 `240 px/s`。每次实际受伤令闪天击和旋风斩剩余 CD 各减少 1 秒。闪天击 CD 8 秒、前摇 0.8 秒，以初始向上速度 `840 px/s` 和水平速度 `180 px/s` 跃起；落地时对自身周围 144 px 造成 20 伤害，并朝前生成 `32×96` 巨型剑气，剑气以 `220 px/s` 前进 300 px，命中造成 20 伤害。旋风斩 CD 4 秒、前摇 0.4 秒，在前摇结束瞬间同时对自身前后各 54 px 造成 20 伤害。闪天击的起跳、空中、落地和旋风斩的前摇、攻击均使用独立状态图片，落地与旋风斩攻击姿态各保留 0.6 秒且不重复判定伤害。
- 菲伦复制体：HP 180，碰撞体 `54×72`，移动速度 `120 px/s`，无碰撞伤害。普通攻击 CD 6 秒、前摇 0.4 秒，向玩家方向发射最长 256 px 的斜向魔法光束并造成 15 伤害；每次光束释放后有 75% 概率再次进入一次普通攻击前摇，直到首次未触发连射后才启动完整 CD。概率流由战斗种子和本次连射序号确定，可由相同种子复现。
- 芙莉莲复制体：HP 350，碰撞体 `54×72`，移动速度 `100 px/s`，无碰撞伤害，初始悬浮于地面上方 48 px。破灭之雷 CD 8 秒、前摇 0.5 秒；前摇结束后立刻恢复其他技能调度，此后每 0.8 秒以当时玩家位置为中心锁定一道 84 px 宽落雷，共 5 道，每道锁定后延迟 0.4 秒生效并造成 20 伤害。地狱业火 CD 8 秒、前摇 0.5 秒，只点燃基础地面而不点燃平台，持续 5 秒并每 0.5 秒造成 5 点接触伤害。普通攻击 CD 3 秒、前摇 0.3 秒，发射最长 256 px 的斜向光束并造成 15 伤害。多个技能同时完成冷却时按破灭之雷、地狱业火、普通攻击的顺序选择。
- 战前对话：芙莉莲“这就是塔顶了吗，感觉攻克难度很一般啊”；水镜恶魔“……”；芙莉莲“哦？复制个体的能力，似乎连个体能力也能完全复制，有点意思”；水镜恶魔“……”。二阶段对话：芙莉莲“这是……我的复制体？”；水镜恶魔“……”；芙莉莲“那么，来试试看吧”。战后对话：水镜恶魔“……”；芙莉莲“呼，真不容易”。Boss 名称演出和全部对话期间暂停战斗。

### 8.3 当前事件内容

- 奥尔登勋爵的舞会：最大 HP +30、随机未拥有魔导书、或 50 金币三选一。
- 半世纪流星雨：随机未拥有魔导书、恢复至最大 HP、或以 50% 概率获得 99 金币三选一。概率结果由楼层事件 RNG 在事件创建时确定，重开同一层保持一致。
- 剑之乡：解析封印获得“勇者之剑”、认可击败魔王之剑获得“真-勇者之剑”，或先完成委托获得 50 金币。
- 勇者之剑和真-勇者之剑是剑之乡专属遗物，不进入商店及普通遗物池。勇者之剑使每次碰撞伤害在统一结算中固定减少 10；真-勇者之剑在玩家实际受到碰撞伤害后，对自身周围 54 px 内所有敌人造成 30 伤害。被黑冲无敌、护盾或减伤完全阻挡的碰撞不触发反击。

## 9. 经济与商店

- 金币主要来自普通战斗，也可由事件或遗物修正。
- 商店出售魔法与遗物。
- 商店必须在购买前显示价格、效果、槽位/收藏状态和无法购买原因。
- 商品数量、价格曲线、刷新和出售机制均为待平衡项。
- 需要商店保底或阶段性价格规则，防止玩家因随机性完全无法完成一次有意义的购买。

## 10. 楼层生成与加载

- 三幕地图主题、竞技场原型、平台尺度、15 层场景表和地图/敌人接口以 `MAP_DESIGN.md` 为权威细化文档。
- 正式楼层不再全部使用单一平地。首版使用基础地面与静态单向平台制造高低差；玩家站在单向平台时可用“下方向 + 跳跃”主动穿台，但基础地面不可穿透。首版不加入斜坡、移动平台或致死深坑。
- 本局开始时生成或确定一个 `runSeed`。
- 每层由 `runSeed + floorIndex + 内容通道` 派生稳定随机结果。
- 楼层必须有可达的出生点、遭遇区域和楼梯出口。
- 生成后先验证路径可达性，再允许玩家进入。
- 与楼梯交互后进行结算、回血、保存必要状态、卸载当前层并加载下一层。
- 不在本局开始时一次性加载全部地图。是否后台预加载下一层由后续性能测试决定。

## 11. UI 与交互

最低界面集合：

- HUD：HP、金币、三个普通魔法槽及各自冷却、终极槽及公共冷却、Boss/楼层进度。
- 普通奖励三选一界面。
- Boss 强力魔法三选一界面。
- 两页构筑界面：第一页显示已学普通/Boss 魔法、三槽普通装备与终极槽，第二页显示本局获得的全部遗物及详细效果。
- 每个可学习魔法使用一张独立创作的方形像素卡面，不得直接裁剪或复用战斗特效图集；同一卡面必须同时出现在奖励三选一、商店魔法书、Tab 魔法页和 HUD 装备槽。Boss 魔法使用金色阶级边框，冷却遮罩覆盖卡面但不能替代权威冷却读数。
- 商店界面。
- 事件选择界面。
- 暂停、失败和胜利结算界面。

当前原型启动时进入 `START` 页面；已有暂退中的本局时显示 `CONTINUE`。`Escape` 打开暂停页，`W/S` 或上下方向键在“重开当前层”和“保存并退出”之间选择，`Enter` 确认，`Escape` 直接恢复游戏。本层重开会回滚该层内产生的 HP、金币、奖励、库存和事件变化。“保存并退出”当前只保留进程内的 `TowerSession` 并返回开始页，可由 `CONTINUE` 恢复；它不是磁盘存档，关闭程序后不会保留。胜利/失败进入结果页，确认后创建全新本局。

开始页提供 `F2` 事件预览、`F3` 商店预览和 `F4` 魔法注册演出预览入口。前两者使用显式首层类型覆盖创建固定种子的特殊楼层，便于验收 NPC 空间交互、三选一、持久事件结果与购买事务；商店预览会把初始金币提高到至少 100。`F4` 创建正常首层并直接为魔法 `1001` 启动纯表现层注册演出，不修改已学或装备状态。正常新局仍使用原有确定性调度概率、保底规则和初始金币。

构筑界面必须显示玩家当前获得的全部魔法和遗物，可在战斗中、奖励界面或战后探索时打开；打开时暂停底层战斗/移动。第一页分为普通魔法与 Boss 魔法两个列表：普通魔法只能明确装备到三个普通槽，Boss 魔法只能装备到终极槽；终极槽未解锁时显示锁定条件。第二页用于浏览本局遗物及其详细效果，遗物不提供装备或卸下操作。`Q/E` 在两页之间切换，`A/D` 浏览当前列表条目，`Tab` 关闭后恢复原阶段。选择奖励只扩充对应已学列表，不自动修改装备。

所有界面必须支持键鼠操作；是否首个版本支持手柄需在技术预研阶段决定。

## 12. 美术与音频方向

### 12.1 美术

- 高幻想魔法世界，强调高塔垂直感、古老遗迹和不同魔法生态。
- 主角气质为冷静、阅历深、以魔法知识解决问题。课程原型阶段允许使用参考性占位设计，最终答辩版本至少保持角色、敌人和可交互物在画面中清楚可辨。
- 敌我攻击通过颜色、轮廓、前摇和特效层级清楚区分。
- 每一幕拥有独立环境主题与敌人语言，第三幕逐步汇聚前两幕机制。

### 12.2 音频

- 关键动作需有独立声音反馈：受击、格挡/无敌、魔法就绪、奖励选择、金币购买、楼梯开启和 Boss 阶段转换。
- 音乐应支持普通楼层、商店/事件、Boss 和胜利/失败状态切换。
- 课程版本优先使用可明确说明来源的占位音乐与音效，避免组员之间出现缺失资源。

## 13. 胜负与局外系统

- **胜利：**击败第三位 Boss。
- **失败：**玩家 HP 降至 0。
- 本局结束后保留哪些内容尚未确定。首个可玩版本应优先验证单局循环，不预设复杂局外天赋树。
- 可保存设置和解锁记录；是否支持中途退出后继续本局需单独决定。

## 14. 首个垂直切片范围

垂直切片不等于完整三幕，而是证明核心循环可玩：

- 1 个可操控角色和完整基础动作。
- 1 个普通战斗楼层模板。
- 至少 2 种普通敌人。
- 至少 4 个可区分的普通魔法，支持学习、三槽普通装备和独立 CD；Boss 1 阶段另验收终极槽。
- 至少 3 个遗物，其中 1 个为风险收益型。
- 普通战金币与魔法三选一。
- 1 个商店楼层。
- 1 个事件原型。
- 1 场 Boss 战与强力魔法三选一。
- 楼梯过层、恢复 50% 已损失 HP、按种子生成下一层。
- 失败和阶段胜利界面。

垂直切片通过后，再扩充到三幕和三位 Boss。

## 16. 术语

- **本局（Run）：**从塔底开始直到失败或击败第三位 Boss 的完整流程。
- **楼层（Floor）：**一次加载的独立游玩单元。
- **遭遇（Encounter）：**楼层内需要完成的战斗、事件、商店或 Boss 内容。
- **已学魔法（Learned Spell）：**本局获得、可参与配置的魔法。
- **魔法槽（Spell Slot）：**三个战斗快捷槽位之一。
- **遗物（Relic）：**获得后在本局持续生效的被动规则修正。
- **强力魔法（Boss Spell）：**Boss 奖励池中的高影响魔法。
