# 《秘法高塔》魔法与遗物体系设计

| 字段 | 内容 |
|---|---|
| 文档版本 | 0.1 |
| 状态 | 设计提案 / 尚未实现 |
| 适用范围 | 三槽魔法、普通/Boss 奖励、商店、事件、遗物构筑 |
| 设计目标 | 在不引入法力值的前提下，用冷却、槽位机会成本、状态联动和风险收益形成多种可玩构筑 |

> 本文是玩法设计，不表示当前代码已经实现。数值是首轮原型基线，必须经过固定种子回归与真人试玩后调整。

## 1. 参考与改编原则

素材首先来自 B 的《设计稿》，并参考《葬送的芙莉莲》中已经出现的魔法名称和意象。动画官方网站的魔法介绍列出了花田、佐尔特拉克、防御魔法、魔力丝线、驭血、模仿、支配、女神三枪、生活魔法及多种元素魔法，可作为课程原型的参考素材：

- [动画《葬送的芙莉莲》官方魔法介绍](https://frieren-anime.jp/special/magic/)
- [Frieren Wiki 魔法列表与出处索引](https://frieren.fandom.com/wiki/Spells)

游戏效果不机械照搬原作，而是遵守以下规则：

1. 每个魔法必须在三槽体系中有明确用途，不能只有名字不同。
2. 每个遗物至少改变一种决策：选什么魔法、何时施放、如何站位或是否承担风险。
3. 课程版不引入法力值、元素反应网、技能树和随机词缀，避免系统失控。
4. 固定保留三个魔法槽。B 稿中“圣杖之证增加一个槽位”的方案会破坏核心约束，本文将其改为三槽之间的冷却联动。
5. 所有效果必须能通过统一伤害、治疗、状态、冷却和触发管线实现与测试。

## 2. 核心体验

理想的一局不应只是把三个最高伤害魔法塞进槽位，而应出现以下选择：

- 一个短 CD 输出、一个控制、一个保命，形成稳定构筑。
- 两个相互触发的魔法加一个风险型遗物，形成爆发构筑。
- 放弃部分伤害，使用区域、恢复和减速进行阵地战。
- 围绕低生命、完美防御、连续命中或首次施法形成专门构筑。
- 根据下一层敌人和 Boss 临时换装，而不是一套配置打完整局。

三个槽位建议分别承担“启动器、核心输出、保险手段”，但不进行强制职业限制。

## 3. 统一术语与标签

### 3.1 魔法学派

学派用于遗物联动和内容表达，不直接决定伤害类型。

| 学派 | 定位 | 典型魔法 |
|---|---|---|
| 基础术式 | 稳定、通用、短 CD | 佐尔特拉克、防御魔法 |
| 元素术式 | 伤害、区域、环境控制 | 火焰、冰箭、石弹、龙卷风 |
| 民间/生活魔法 | 条件式收益与特殊工具 | 花田、热茶、清洁、酸葡萄 |
| 女神魔法 | 净化、防护、恢复 | 女神加护、女神三枪、解咒 |
| 魔族魔法 | 高上限、控制或代价 | 驭血、支配、模仿、魔力丝线 |

### 3.2 机制标签

一个魔法可以拥有多个标签。遗物只读取标签，不读取魔法名称。

`直接伤害`、`投射物`、`区域`、`持续`、`控制`、`防御`、`位移`、`恢复`、`诅咒`、`生活`、`血魔法`、`Boss魔法`。

### 3.3 标准状态

| 状态 | 统一规则 |
|---|---|
| 缓速 | 只降低水平移动速度，不改变攻击前摇；同名缓速取最高值并刷新时间 |
| 束缚 | 禁止水平移动，但仍允许已经开始的攻击继续；Boss 按控制抗性缩短 |
| 硬直 | 中断当前动作；Boss 只接受短硬直，不接受长时间击倒 |
| 破防 | 受到的直接伤害提高 20%；不放大 HP 消耗、环境伤害和遗物追加伤害 |
| 解析 | 下一次直接伤害消耗该标记，额外造成 25% 伤害并获得轻微追踪修正 |
| 护盾 | 先于 HP 承伤；不同来源护盾相加，但不超过最大 HP 的 30% |
| 加护 | 免疫新施加的负面状态；已经存在的负面状态需由净化移除 |
| 诅咒 | 属于负面状态，可被解咒；普通防御魔法不能直接格挡其持续效果 |

## 4. 全局战斗规则

### 4.1 施法事务顺序

每次施法严格按以下顺序结算：

1. 检查槽位、角色状态、冷却和施法条件。
2. 支付 HP 等施法代价；代价不能被增伤、减伤或护盾修改。
3. 启动该魔法冷却。
4. 创建投射物、区域、护盾或状态。
5. 命中时进入统一伤害/治疗管线。
6. 应用控制或负面状态。
7. 触发“施法后”“造成伤害后”“命中后”遗物。
8. 处理击杀；追加伤害不得重新触发自身或同一条触发链。

### 4.2 伤害与冷却边界

```text
最终伤害 = round((基础伤害 × 来源倍率 × 目标倍率) - 固定减伤)
最终冷却 = clamp((基础冷却 + 固定修正) × 冷却倍率, 基础冷却 × 0.5, 基础冷却 × 2.0)
```

- 普通增伤合计上限为 +100%，特殊 Boss 机制可单独突破。
- 受到伤害倍率限制在 50%～200%。
- 单个魔法一次施放对同一敌人只能产生一次“命中”触发；持续伤害的每一跳只算“伤害”，不重复算“施法”。
- 自动施放和回响施放必须标注来源，不能触发另一次自动施放或回响。

### 4.3 控制抗性

- 普通敌人承受 100% 控制时长。
- 精英承受 60%。
- Boss 承受 35%，单次硬控制最长 1 秒。
- 同一目标在 4 秒内再次受到硬控制，持续时间额外减半；4 秒未受控后恢复。
- 支配对 Boss 不改变阵营，只造成 0.8 秒硬直与 4 秒破防。

### 4.4 战斗恢复限制

为了避免玩家留下最后一个敌人无限回血：

- 只有遭遇仍处于战斗状态且至少有一个活敌人时，战斗恢复魔法才生效。
- 每层由魔法产生的 HP 恢复总量不超过最大 HP 的 40%。
- 击败最后一个敌人后立即清除持续治疗区域，但保留掉落物和地图探索。
- 楼梯恢复不计入该上限。

### 4.5 战斗中换装

装备栏仍可随时打开并暂停底层战斗。为了防止轮换全部已学魔法绕过冷却：

- 冷却状态属于“魔法”，不属于槽位；换出后仍继续计时。
- 战斗中换入魔法时，剩余冷却至少为其基础冷却的 50%。
- 同一魔法不能同时装备在多个槽位。
- 战斗外换装不施加 50% 起始冷却。

## 7. 遗物体系

### 7.1 通用规则

- 每种遗物每局只能获得一次，不设计叠加等级。
- 遗物触发分为：战斗开始、施法、命中、伤害、受击、完美防御、击杀、楼层结算和经济事件。
- 追加伤害带有 `RelicSecondary` 来源，不再次触发“造成伤害后”的追加伤害遗物。
- 自动施法不启动装备槽冷却，也不触发“首次主动施法”遗物。
- 同时触发时按“保命 → 状态 → 冷却 → 伤害 → 经济”顺序结算。

## 8. 推荐构筑原型

构筑不是预设职业，而是验证内容之间确实存在多条可行路线。

### 8.1 高频佐尔特拉克

- 魔法：普通佐尔特拉克、防御魔法、高速移动。
- 遗物：紫葡萄发饰、腐败贤老封印碎片、圣杖之证。
- 节奏：高速移动增伤 → 连续投射 → 完美防御缩短冷却。
- 弱点：缺少恢复，面对多方向攻击压力大。

### 8.2 花田阵地战

- 魔法：催生花田、酸葡萄、火焰魔法。
- 遗物：苍月草的花、勇者的花冠、古旧书签。
- 节奏：以花田减速和恢复维持安全区，在破防区域内叠加燃烧。
- 弱点：敌人离开区域或 Boss 高频位移时输出下降。

### 8.3 驭血爆发

- 魔法：驭血魔法、女神加护、防御魔法或热茶。
- 遗物：暗黑龙的角、镜莲华戒指、僧侣圣典。
- 节奏：加护 → 驭血 → 回响形成高爆发，再用防御/恢复控制风险。
- 弱点：连续失误会迅速进入低血低伤状态。

### 8.4 控制处刑

- 魔法：魔力丝线、捕鸟魔法、大致能斩断一切。
- 遗物：服从天平、芙拉梅手记、魔力探测镜片。
- 节奏：开局布置 → 拉近/束缚 → 短距高伤斩击。
- 弱点：Boss 控制抗性高，必须依赖站位而非永久控制。

### 8.5 女神反击

- 魔法：防御魔法、女神加护、女神三枪。
- 遗物：僧侣圣典、破损防御术式、勇者花冠。
- 节奏：完美防御创造窗口，用加护覆盖危险阶段并以三枪回血。
- 弱点：需要读招，主动清怪速度中等。

### 8.6 探索经济流

- 魔法：寻找饰品、看穿伪装、一个稳定输出魔法。
- 遗物：一级魔法使徽章、宝箱怪舌片、芙拉梅手记。
- 节奏：牺牲一个战斗槽换取金币、信息与更可控的奖励。
- 弱点：早期战斗能力低，必须利用信息与商店补强。

## 9. 获取、商店与奖励规则

### 9.1 普通战奖励

- 从普通魔法池提供三个未拥有候选。
- 三个候选尽量不全是同一功能标签；若可能，至少包含一个直接伤害或一个生存工具。
- 玩家已有全部普通魔法时改发 15 金币，保持当前原型行为。
- 不引入重复魔法升级，避免第一版同时维护等级和平衡矩阵。

### 9.2 Boss 奖励

- 仅从 Boss 魔法池提供三个未拥有候选。
- 同一次选择中不同时出现功能完全重叠的两个纯伤害大招。
- 第三个 Boss 奖励仍可展示和选择，然后通过楼梯或结算交互进入胜利，保持奖励流程一致。

### 9.3 商店

每个商店生成三个商品：

1. 至少一本未拥有的普通魔法。
2. 至少一件未拥有的遗物。
3. 一个魔法/遗物随机位。

建议基础价格：普通魔法 20～35 金币，扩展/事件魔法 35～50，遗物 35～60。价格随幕数最多增加 20%，不随玩家当前金币动态定价。

### 9.4 事件

- 事件可直接给予扩展魔法、遗物或一次重投机会。
- “奥尔登勋爵的舞会”保留三项：最大 HP +30、随机未拥有魔法、50 金币。
- 生活魔法优先通过事件和商店出现，避免普通战奖励频繁给出暂时无战斗价值的选择。

## 10. 数值预算

### 10.1 普通魔法

| 类型 | 建议范围 |
|---|---|
| 短 CD 直接伤害 | 18～25 伤害，1.2～2 秒 CD |
| 带控制的伤害 | 20～35 伤害，3.5～6 秒 CD |
| 重型爆发 | 38～50 伤害，6～9 秒 CD，带射程/前摇风险 |
| 硬控制 | 普通敌人 1～2.5 秒，8～12 秒 CD |
| 恢复 | 单次 10～20 HP，并受每层 40% 最大 HP 总上限约束 |
| 增伤状态 | +15%～25%，覆盖率不应超过约 50% |

### 10.2 遗物

- 无条件伤害增益通常不超过 15%。达到 20% 时必须附带风险或条件。
- 单件遗物每场战斗的期望追加伤害约为普通敌人最大 HP 的 20%～35%。
- 冷却缩减单次通常为 0.5～1 秒，总下限受基础 CD 50% 限制。
- 遗物不能直接让 Boss 永久失去行动能力。
- 经济遗物应提高选择空间，而不是保证每个商店都能买空。

## 11. 内容优先级

### 第一批：形成可玩闭环

- 12 个核心普通魔法。
- 6 个 Boss 魔法：2001～2006。
- 12 件首发遗物。
- 五种推荐构筑中至少四种可以通过相同测试战斗房。

### 第二批：增加原作气质与重玩价值

- 6 个生活/探索扩展魔法。
- 破灭之雷、结界破坏。
- 4 件扩展遗物。
- 新事件对扩展魔法和遗物的定向获取。

不要在第一批完成前增加魔法升级、遗物等级、随机词缀或第四槽。

## 12. 实现前的数据契约建议

魔法定义至少需要以下稳定字段：

```text
id, localizationKey, school, tags, targeting,
baseCooldown, castTime, damageSpec, costSpec,
statusEffects, duration, triggerPolicy, rewardPool
```

遗物定义至少需要：

```text
id, localizationKey, tags, trigger,
condition, operation, priority,
perTargetCooldown, perCombatLimit, recursionPolicy
```

状态和效果使用稳定 ID，不让 UI、商店或遗物通过中文名称判断逻辑。所有内容加载后必须校验：引用的魔法、状态、标签与图标存在，奖励池至少能生成合法候选。

## 13. 必须覆盖的测试场景

1. 三个槽位冷却相互独立，换装不能刷新冷却。
2. HP 代价先于驭血伤害计算，且不会把玩家直接扣至 0。
3. 花田治疗受每层上限限制，最后敌人死亡后停止治疗。
4. Boss 控制抗性和连续控制衰减生效。
5. 镜莲华回响、勇者之剑和追加伤害不会递归触发。
6. 苍月草自动花田不占槽、不启动冷却、不触发首次主动施法。
7. 圣杖之证始终保持三个槽位，并且每槽每场最多触发一次。
8. 固定种子下奖励、商店、重投和事件结果可重复。
9. 每一种商店魔法都存在战斗定义，并能完成购买、装备和施放。
10. 至少四种推荐构筑可击败同一标准战斗房，且不存在单一必选组合。

## 14. 需要试玩决定的参数

- 女神加护采用 6 秒/14 秒还是当前原型的 8 秒/10 秒。
- 花田的治疗上限是最大 HP 的 30% 还是 40%。
- 战斗中换入魔法的最低起始冷却是 50% 还是完整冷却。
- Boss 控制倍率采用 35% 还是 40%。
- 商店价格和每层平均金币收入。
- 诛魔魔法对“魔族”敌人的额外伤害是否会使特定 Boss 过于简单。

这些参数应通过数据配置调整，不应写死在 UI 或敌人代码中。
